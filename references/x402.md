# x402: pay and serve paid HTTP endpoints

Source of truth: `packages/x402/src/` (`@blockchain0x/x402`). x402 is USDC on
Base only; cross-network is rejected (`AGENTS.md` §1.1). The package exports the
wire primitives at the root and framework adapters at subpaths:

| Import                              | Surface                                                                                |
| ----------------------------------- | -------------------------------------------------------------------------------------- |
| `@blockchain0x/x402`                | `buildPaymentHeader`, `parsePaymentHeader`, `parse402Response`, `X402WireError`, types |
| `@blockchain0x/x402/client`         | `createX402Client` (a fetch wrapper)                                                   |
| `@blockchain0x/x402/server/fastify` | `createX402Plugin`                                                                     |
| `@blockchain0x/x402/server/express` | `createX402Middleware`                                                                 |

## Wire format

```
X-Payment: <scheme>:<base64(payload)>
```

The only scheme is `exact-usdc`, so the header is
`X-Payment: exact-usdc:<base64>`. The base64 body is a JSON payment payload.
`buildPaymentHeader` encodes it and `parsePaymentHeader` decodes it; an unknown
scheme or malformed payload throws `X402WireError`
(`header.missing` / `header.unknown_scheme` / `header.payload_malformed`).

```ts
import { buildPaymentHeader, parsePaymentHeader } from '@blockchain0x/x402';

const header = buildPaymentHeader({
  scheme: 'exact-usdc',
  version: 1,
  paymentRequestId: 'pr_01HQ',
  txHash: '0x' + '0'.repeat(64),
  payerAddress: '0xabc0000000000000000000000000000000000000',
  amountUsdc: '0.10',
  network: 'testnet',
});
const payment = parsePaymentHeader(header); // -> ExactUsdcPayment
```

## Pay an endpoint (consume a 402) - `pay_bills`

`createX402Client({ sdk })` returns a `fetch`-compatible function. When a request
gets a 402 challenge it pays the requirement, retries once with the `X-Payment`
header attached, and returns the paid response.

```ts
import { createClient } from '@blockchain0x/node';
import { createX402Client } from '@blockchain0x/x402/client';

const client = createClient({ apiKey: process.env.BLOCKCHAIN0X_API_KEY! });
const fetchWithPay = createX402Client({ sdk: client });

const res = await fetchWithPay('https://api.vendor.com/llm-query', {
  method: 'POST',
});
```

## Serve a paid endpoint (issue 402s) - `receive_money`

The server adapters take `{ sdk, pricing }` (`AdapterOptions`), where `pricing`
maps `"<METHOD> <path>"` to `{ amountUsdc, payToAddress, network? }`. On a paid
request the adapter verifies the `X-Payment` header and settles the matched
payment request exactly once via `sdk.paymentRequests.settle`.

```ts
const pricing = {
  'POST /llm-query': {
    amountUsdc: '0.10',
    payToAddress: '0xabc0000000000000000000000000000000000000',
  },
};
```

Mount with `createX402Plugin({ sdk: client, pricing })` on Fastify, or
`createX402Middleware({ sdk: client, pricing })` on Express. See
`packages/x402/src/server/` for the full `AdapterOptions`
(`defaultNetwork`, `maxAgeSeconds`).

## Cross-language + cross-network

The wire format is identical across the language SDKs, so a Python service
accepts payments from a Node payer and vice versa. A testnet payer cannot pay a
mainnet resource (and the reverse): the network is pinned by the key prefix and
the requirement (`AGENTS.md` §1.1, [scopes-and-keys.md](scopes-and-keys.md)).
